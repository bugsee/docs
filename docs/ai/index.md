# Bugsee for AI

Connect Bugsee to your AI coding assistants and workflows. Install the Bugsee plugin, drop in agent skills, or connect the MCP server — then let your assistant set up Bugsee SDKs, triage crashes and bug reports, and keep stack traces readable, all without leaving your editor.

## Tools for your agents

| Tool | What it does |
|------|--------------|
| **[Plugins](/ai/plugins)** | One-step install for Claude Code, Cursor, Codex, and other agents. Bundles Bugsee's agent skills and connects the MCP server. |
| **[Agent Skills](/ai/agent-skills)** | Instruction sets that teach your assistant to install and configure a Bugsee SDK in your project — no manual setup required. |
| **[MCP Server](/mcp)** | Connects your assistant directly to your Bugsee issues (crashes, errors, bug reports) and builds (size, dependencies, vulnerabilities) for live, in-editor debugging. |

## What you can do

- **Add Bugsee to an app** — detect the platform and run a complete SDK setup (install → initialize → configure) for iOS, Android, Flutter, React Native, Unity, .NET/MAUI, Xamarin, Cordova, or Kotlin Multiplatform.
- **Fix issues from Bugsee** — pull a crash, error, or bug report's full context (stack trace, breadcrumbs, network, console logs) into your editor and root-cause it against your code.
- **Track build health** — check build size, what a release added or removed, size/dependency/timing regressions, and dependency vulnerabilities.
- **Keep traces readable** — upload dSYMs, JavaScript source maps, and Android mapping files so production stack traces resolve to your source.

## Get started

- New to this? Start with **[Plugins](/ai/plugins)** for a one-step install in your assistant.
- Want to add a single skill by hand, or use a client without plugin support? See **[Agent Skills](/ai/agent-skills)**.
- Want your assistant to read live issue data? Connect the **[MCP Server](/mcp)**.
